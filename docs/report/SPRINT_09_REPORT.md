## Ringkasan Eksekutif

Sprint 9 mengimplementasikan sistem notifikasi end-to-end tanpa perubahan schema database. Tabel `notifications` dari Sprint 5 dipakai kembali untuk menyimpan alert API inactive, error rate, dan response time.

Scope yang selesai:

- Backend notification service, repository support, route protected, dan scheduler 5 menit.
- Alert generator untuk lima aplikasi: Marketplace, POS, SupplierHub, LogistiKita, dan SmartBank.
- Frontend notification bell di dashboard Admin, User, dan Monitoring.
- Mark single notification dan mark all visible notifications sebagai read.
- Test TDD backend/frontend, build, Docker checklist, dan smoke test API.

## Backend

### Package `internal/notification`

Package baru `internal/notification` berisi service utama untuk:

- List notifikasi dengan pagination default `page=1`, `limit=10`, max `50`.
- Visibility per role:
  - `admin_gateway`: semua notifikasi.
  - `monitoring_user`: semua notifikasi, read-only dari sisi dashboard tetapi endpoint read tetap tersedia untuk notifikasi visible.
  - `app_user`: hanya notifikasi dengan `app_name` sesuai token.
- Mark read single notification dengan validasi visibility.
- Mark all visible notifications.
- Alert generation dengan dedup `app_name + type` selama 24 jam.

### Alert Rules

| Type | Rule |
| --- | --- |
| `api_inactive` | Tidak ada request dalam 7 hari terakhir. |
| `error_rate` | Window 1 jam, minimal 10 request, non-2xx rate lebih dari 10%. |
| `response_time` | Window 1 jam, minimal 10 sample durasi, rata-rata durasi lebih dari 300ms. |

Scheduler berjalan saat backend start, melakukan run awal, lalu interval 5 menit.

### API Contract

#### `GET /notifications`

Protected endpoint. Query:

- `page`: default `1`.
- `limit`: default `10`, max `50`.

Response sukses:

```json
{
  "status": "success",
  "data": {
    "notifications": [
      {
        "id": 1,
        "created_at": "2026-06-21T00:00:00Z",
        "app_name": "Marketplace",
        "type": "api_inactive",
        "message": "API Marketplace tidak aktif selama lebih dari 1 minggu.",
        "is_read": false
      }
    ],
    "unread_count": 1,
    "page": 1,
    "limit": 10
  }
}
```

#### `POST /notifications/read`

Protected endpoint. Body single:

```json
{ "notification_id": 1 }
```

Body mark all visible:

```json
{ "all": true }
```

Response sukses:

```json
{
  "status": "success",
  "data": {
    "unread_count": 0
  }
}
```

Error behavior:

- `400` untuk JSON invalid atau body yang tidak memilih tepat satu dari `notification_id` / `all=true`.
- `401` tanpa token atau token invalid.
- `403` role tidak valid.
- `404` notification ID tidak ada atau tidak visible untuk role/app tersebut.

## Risiko Tersisa dan Handoff ke Sprint 10

- Notification bell memakai polling 60 detik, belum WebSocket/SSE.
- Monitoring user dapat melakukan mark-read untuk notifikasi visible; jika produk menginginkan strict read-only mutation untuk monitoring, perlu perubahan contract eksplisit.
- Alert generator bergantung pada data `request_logs`; data seed yang minim dapat memicu `api_inactive` saat startup, sesuai rule Sprint 9.
- Belum ada halaman arsip/riwayat penuh untuk notifikasi di luar dropdown.
